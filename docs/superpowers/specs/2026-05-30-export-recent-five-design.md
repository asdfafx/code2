# 导出页最近五次多选导出设计

**日期：** 2026-05-30

**目标**

让导出页支持从当前用户最近五次导入记录中选择一个或多个批次，并在点击 CSV、HTML、JSON 导出时，为每个已选批次分别下载一个文件。

**当前状态**

- 导出页 [app/templates/export.html](f:/Code%202/app/templates/export.html) 只展示最近一次导入记录。
- 后端 [app/routes/export.py](f:/Code%202/app/routes/export.py) 已支持通过 `import_id` 导出单个批次；未传 `import_id` 时默认导出最近一次。
- 日志批次列表接口 [app/routes/logs.py](f:/Code%202/app/routes/logs.py) 已有 `/api/logs/list`，可按时间倒序返回当前用户的导入记录。

**方案概述**

复用现有 `/api/logs/list` 和 `/api/export/{format}` 接口，不新增数据表，也不新增批量导出后端接口。导出页在加载时请求最近五次导入记录，并把它们渲染为复选列表。用户可选择一个或多个批次；点击任一导出格式按钮后，前端按选中顺序逐个调用对应导出接口，每个 `import_id` 触发一次独立文件下载。

这样做的原因是：

- 后端现有单批次导出逻辑已经完整，复用成本最低。
- 用户明确要求“多个文件分别下载”，前端串行触发最贴合该行为。
- 选择范围限定为最近五次，浏览器连续下载的数量可控。

**界面设计**

导出页新增“最近五次检测结果”选择区，展示最多 5 条导入记录。每条记录包含：

- 复选框
- 文件名
- 导入时间
- 总记录数
- 已解析数

交互规则：

- 默认勾选最新的一条记录。
- 若只有 1 条记录，页面行为与现有基本一致，但仍使用复选框呈现。
- 若没有记录，显示空状态并禁用三个导出按钮。
- 若用户未勾选任何记录就点击导出，页面提示“请至少选择一条检测结果”。
- 导出过程中按钮临时禁用，避免重复触发。
- 导出完成后恢复按钮，并提示成功数量；若部分失败，提示失败批次。

**前端实现边界**

修改 [app/templates/export.html](f:/Code%202/app/templates/export.html)：

- 用 `/api/logs/list?per_page=5` 替代当前仅请求 `/api/logs/last` 的展示逻辑。
- 新增记录列表渲染函数，维护选中的 `import_id` 集合。
- 把 `exportCSV`、`exportHTML`、`exportJSON` 收敛为共用的批量导出函数，参数仅区分格式和默认扩展名。
- 对每个选中的 `import_id` 发起一次 `POST /api/export/{format}` 请求，请求体包含该 `import_id`。
- 下载文件名优先使用响应头中的服务端文件名；若浏览器端仍需兜底命名，则带上原始文件名或批次 ID，避免多个下载重名。

前端不负责合并文件，也不新增 ZIP 打包逻辑。

**后端实现边界**

后端保持单批次导出语义不变，但补一个最小增强：

- 当请求明确传入 `import_id` 时，继续只导出该批次。
- 生成下载文件名时带上导入批次信息，减少批量下载时文件名重复，例如在时间戳前加入导入文件名净化结果或 `import_id`。

不新增批量接口，不接受数组形式的 `import_id`，避免和“多个文件分别下载”的交互目标冲突。

**数据流**

1. 用户进入导出页。
2. 前端请求 `/api/logs/list?per_page=5`。
3. 页面展示最近五次导入记录，并默认勾选最新一条。
4. 用户勾选一个或多个记录。
5. 用户点击某个导出格式按钮。
6. 前端按选中顺序，逐个向 `/api/export/csv`、`/api/export/html` 或 `/api/export/json` 发送包含 `import_id` 的请求。
7. 每个成功响应都会触发一次浏览器下载。
8. 页面汇总成功/失败结果并提示用户。

**错误处理**

- `/api/logs/list` 返回空：显示“暂无检测记录”，禁用按钮。
- 导出时未选中记录：直接前端提示，不发请求。
- 单个批次导出失败：记录该批次失败信息，继续下一个批次，最终汇总提示。
- 登录失效或接口 401：沿用现有页面错误提示方式。
- 某条记录已被删除但仍在选择列表中：该条请求失败后提示该批次导出失败，不影响其他下载。

**测试策略**

优先补后端测试，确保单批次导出在显式 `import_id` 下行为稳定，并验证文件命名增强。前端因为当前项目未见成体系前端测试基建，可先做最小后端自动化测试，加上人工验证导出页交互。

建议测试覆盖：

- `/api/export/csv` 传入指定 `import_id` 时只导出该批次。
- `/api/export/html` 传入指定 `import_id` 时返回对应批次报告。
- `/api/export/json` 传入指定 `import_id` 时 `import_info` 与结果条目匹配。
- 文件名包含批次识别信息，便于多文件下载区分。
- 导出页能加载最近五次记录、默认选中最新一条、可多选并逐个下载。

**涉及文件**

- 修改：[app/templates/export.html](f:/Code%202/app/templates/export.html)
- 修改：[app/routes/export.py](f:/Code%202/app/routes/export.py)
- 新增：测试文件，位置按现有测试结构补齐

**非目标**

- 不支持一次请求导出多个批次并合并内容。
- 不生成 ZIP。
- 不扩展到“最近五次”之外的任意历史筛选。
- 不改动日志导入、分析、存储模型。
